# FreshTable 배포 FAQ

## Q1. GitHub 없이도 되나요?
네. 가능합니다. 다만 초보자에게는 GitHub 연결이 가장 쉽고, GitHub 없이 진행할 때는 Vercel CLI와 Railway CLI를 사용하는 방식이 더 현실적입니다.

## Q2. 왜 프론트와 백엔드를 따로 배포하나요?
Vercel은 정적 프론트엔드와 SPA/PWA 배포에 매우 편하고, Railway는 Node.js 서버와 PostgreSQL 운영이 편하기 때문입니다.

## Q3. `VITE_API_URL` 이 꼭 필요한가요?
네. 프론트엔드가 어느 백엔드로 API 요청을 보낼지 알아야 해서 필수입니다.

## Q4. CORS 오류는 왜 생기나요?
브라우저 보안 정책 때문에, Railway 백엔드는 허용된 프론트 주소만 받아야 합니다. 그래서 `CORS_ORIGIN` 에 Vercel 주소를 정확히 넣어야 합니다.

## Q5. 앱 설치가 안 보여요.
- Android Chrome: 설치 프롬프트 또는 브라우저 메뉴 확인
- iPhone Safari: `공유` → `홈 화면에 추가`
- HTTPS가 아닌 경우 설치가 제한될 수 있습니다.

## Q6. 샘플 데이터도 바로 보이나요?
시드 데이터를 넣으면 바로 보입니다. Railway에 올라간 뒤 DB가 비어 있다면 seed를 한 번 실행해야 합니다.

## Q7. 재배포는 언제 해야 하나요?
환경변수를 바꿨다면 재배포가 필요합니다. 특히 Vercel 문서 기준으로 환경변수 변경 사항은 기존 배포에 즉시 반영되지 않고 새 배포에 반영됩니다.
